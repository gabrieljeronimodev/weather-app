# Weather App

A simple weather application that allows users to search for a city or country and view real-time weather information. The app displays temperature, localization, humidity, wind speed, and weather conditions based on the user's input.

**Link to project:** https://gabrieljeronimodev.github.io/weather-app/

## How It's Made:

**Tech used:** HTML, CSS, JavaScript

This project was built using semantic HTML for structure, CSS for layout and styling, and Vanilla JavaScript for functionality and API integration.

The application fetches real-time weather data from [OpenWeather](https://openweathermap.org/) API using `fetch` and `async/await`. I implemented error handling to manage invalid searches or failed API requests, ensuring a better user experience.

JavaScript was used to dynamically update the DOM with the returned data, including temperature, humidity, wind speed, and weather conditions. I also structured the code into small, reusable functions to keep it organized and maintainable.

One of the main goals of this project was to strengthen my understanding of:
- Asynchronous JavaScript
- API consumption
- Event handling
- DOM manipulation
- Clean and readable code structure
  
## Lessons Learned

Building this project helped me better understand how front-end applications communicate with external APIs and handle asynchronous data.

One of the biggest learning moments was working with `async/await` and understanding how to properly handle errors when dealing with API requests. It was rewarding to see real-time data being fetched and dynamically rendered on the screen based on user input.

I also improved my understanding of DOM manipulation, event handling, and structuring JavaScript code into smaller, organized functions. Separating the data-fetching logic from the UI rendering logic made the project easier to maintain and scale.

This project reinforced the importance of writing clean, readable code and thinking about user experience, especially when handling invalid inputs or unexpected API responses.

Completing this application gave me more confidence in building interactive front-end projects and working with real-world data.
